<objective>
Create the WayfindingManager class with LLM tool definitions and handlers, extend MapManager with destination highlight rendering, and add new EventTypes for display dispatch.

Purpose: This is the core wayfinding logic -- tool definitions that the LLM will call, handlers that bridge between the LLM and the navigation/map layers, and the visual highlight rendering for showing destinations on the map. Without this, the robot cannot process "where is X?" or "take me to X" requests.

Output: WayfindingManager class, MapManager.render_map_with_highlight(), two new EventTypes, all verified with TDD tests.
</objective>

   - Add a _poi_positions: dict[str, dict] = {} cache to MapManager.__init__
   - Subscribe to POI_LIST_UPDATED in start() to populate _poi_positions from marker data (each marker has name and pose.position.x/y or flat x/y)
   - Add _on_poi_list_updated(data) handler: iterates data["markers"], extracts name and position coords, stores in _poi_positions
   - render_map_with_highlight: copies the base render_map logic, but before drawing the robot arrow, checks _poi_positions.get(poi_name).    - Use self._cfg.highlight_color and self._cfg.highlight_radius_px as defaults.

2. Implement WayfindingManager._handle_query_location(args):
   - Extract location_name from args
   - Call self._poi_kb.resolve(location_name)
   - If None: return {"found": False, "verbal": f"I don't know where {location_name} is. I know about: {', '.join(self._poi_kb.list_locations()[:5])}."}
   - If found: call self._map.render_map_with_highlight(poi_name) to get PNG bytes
   - Emit DISPLAY_MAP event with {"png": png_bytes, "highlighted_poi": poi_name}
   - Return {"found": True, "poi_name": poi_name, "verbal": f"I found {location_name} on the map!"}

3. Implement WayfindingManager._handle_navigate_to(args):

   - Else: return {"started": False, "verbal": f"Sorry, I couldn't navigate to {poi_name}."}
