- Server sends JSON: {"type":"tts","text":"Hello"}

  <behavior>
    - write(pcmData, offset, length): enqueues PCM bytes for background playback (does not block caller)
    - start(): creates AudioTrack (24kHz, mono, PCM16, MODE_STREAM), calls play(), starts background HandlerThread
    - stop(): stops and flushes AudioTrack, does NOT release (reusable for next TTS utterance)
    - release(): stops + releases AudioTrack, quits HandlerThread
    - setVolume(vol: Float): calls audioTrack.setVolume(vol) clamped to 0.0..1.0
    - handleBinaryFrame(data: ByteArray): if data[0]==0x05, calls write(data, 1, data.size-1); ignores other types
    - handleBinaryFrame: returns false for empty data or unknown type bytes
  </behavior>
  <action>
    1. Write TtsAudioPlayerTest.kt first with JUnit 4 tests. Since AudioTrack is an Android SDK class that cannot be instantiated in JVM unit tests, test the pure logic:
       - handleBinaryFrame routing: verify it identifies 0x05 type byte and extracts PCM payload
       - handleBinaryFrame: verify empty array returns false, non-0x05 type returns false
       - Volume clamping: verify setVolume clamps to 0.0..1.0 range

    For the AudioTrack integration, use a testable design: extract the frame parsing and routing logic into a companion object or separate function that can be tested without Android dependencies. The AudioTrack write() call should be behind an interface or lambda that tests can mock.

    2. Create TtsAudioPlayer.kt:
       - Constructor takes no args (AudioTrack created in start())
       - start(): creates AudioTrack with Builder pattern:
         - AudioAttributes: USAGE_MEDIA, CONTENT_TYPE_SPEECH
         - AudioFormat: ENCODING_PCM_16BIT, sampleRate=24000, CHANNEL_OUT_MONO
         - TransferMode: MODE_STREAM
         - BufferSize: getMinBufferSize(24000, CHANNEL_OUT_MONO, PCM_16BIT) * 4
         - Calls audioTrack.play()
         - Creates HandlerThread("TtsPlayback") for background writes
       - write(pcmData, offset, length): posts audioTrack.write(pcmData, offset, length) to HandlerThread
       - stop(): posts { audioTrack.stop(); audioTrack.flush() } to HandlerThread, sets isSpeaking flag false
       - release(): audioTrack.release(), handlerThread.quitSafely()
       - setVolume(vol): audioTrack.setVolume(vol.coerceIn(0f, 1f))
       - handleBinaryFrame(data): parses type byte, routes 0x05 to write

    CRITICAL: AudioTrack.write() MUST run on the HandlerThread, NOT on the OkHttp callback thread. OkHttp's dispatch thread will block if write() blocks, causing WebSocket frame queueing.

    3. Run tests to verify all pass.
