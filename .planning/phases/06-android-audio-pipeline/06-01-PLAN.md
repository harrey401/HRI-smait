  <behavior>
    - adapt8chTo6chCaeFormat: 16 bytes input (one 8ch frame) produces 24 bytes output (one 6ch 32-bit frame)
    - adapt8chTo6chCaeFormat: each 4-byte slot has format [0x00, channel_id, pcm_lo, pcm_hi] with IDs 1..6
    - adapt8chTo6chCaeFormat: source channel mapping is ch0-3 (mics) + ch6,ch7 (refs) from 8ch input
    - extract4chRaw: 16 bytes input produces 8 bytes output (ch0-ch3 only, S16LE interleaved)
    - sendAudio: prepends 0x01 type byte to CAE beamformed audio data
    - sendRaw4ch: prepends 0x03 type byte to extracted 4ch raw audio
    - sendDoaAngle: sends JSON text {"type":"doa","angle":N,"beam":N} — NOT binary
    - sendDoaAngle: does NOT prepend any binary type byte (text frame, not binary frame)
    - CAESetRealBeam(0) called after CaeCoreHelper init to enable continuous output
    - pcmListener: calls adapt8chTo6chCaeFormat then writeAudio, AND calls extract4chRaw then sendRaw4ch
    - onAudio callback: sends CAE beamformed output via sendAudio (0x01) — NOT bypassed
  </behavior>
  <action>
    1. Write CaeAudioManagerTest.kt first with JUnit 4 tests covering all behaviors above. Mock WebSocket as a simple recording wrapper to capture sent bytes/strings. Test the pure functions directly (adapt8chTo6chCaeFormat, extract4chRaw) and verify frame format of send methods.

    2. Create CaeAudioManager.kt adapted from the cae-work-march2 branch version (available at /home/gow/.openclaw/workspace/projects/smait-jackie-app/ on branch cae-work-march2). Apply these three critical fixes:

    FIX 1 — Channel adapter (AUD-02): Replace `[0x00, 0x00, pcm_lo, pcm_hi]` with `[0x00, ch_id, pcm_lo, pcm_hi]` where ch_id = 1..6. The branch used zero for byte index 1 which is why CAE onAudio never fired. Reference implementation: docs/hardware-sdk/CAEDemoAIUI-4 MIC/CAEDemoAIUI/app/src/main/java/com/voice/caePk/CaeOperator.java adapeter4Mic32bit().

    FIX 2 — DOA format (AUD-04): Replace binary ByteBuffer DOA send (type 0x03 + float) with JSON text frame: `webSocket?.send(JSONObject().put("type","doa").put("angle",angle).put("beam",beam).toString())`. Remove DOA_TYPE constant. The server's _handle_text() parses JSON; binary 0x03 would collide with AUDIO_RAW.

    FIX 3 — Remove CAE bypass (AUD-01, AUD-03): Remove the "BYPASS CAE" mono extraction + 16x gain code from pcmListener. Instead:
    - pcmListener calls adapt8chTo6chCaeFormat then caeCoreHelper.writeAudio (feeds CAE)
    - pcmListener also calls extract4chRaw then sendRaw4ch (sends 0x03 raw 4ch)
    - onAudio callback sends CAE output via sendAudio (sends 0x01 beamformed)

    Keep: copyAssetsIfNeeded(), start(ws), stop(), cleanup(), ALSA config constants, CAESetRealBeam(0) after init.

    Make adapt8chTo6chCaeFormat, extract4chRaw, and the frame-building parts of sendAudio/sendRaw4ch/sendDoaAngle accessible for testing. Either make them internal/package-private or extract as companion object functions. Prefer making them `internal` so tests in the same module can access them.

    3. Run tests to verify all pass.
